# Адаптер QwenTTS для SkyrimNet

---

Этот проект реализует интеграцию локального быстрого движка синтеза речи **QwenTTS** (на базе [qwentts.cpp](https://github.com/ServeurpersoCom/qwentts.cpp)) в качестве бэкенда для SkyrimNet.

Адаптер эмулирует стандартное API SkyrimNet (`/tts_to_audio/`, `/health`, `/create_and_store_latents`), обеспечивая бесшовную работу клиента.

---

## Основные особенности движка

* **Локальный и быстрый**: Написан на C++17 (GGML), работает полностью на CPU (или GPU через CUDA/Vulkan) без тяжелых зависимостей (PyTorch/Transformers).
* **Клонирование голоса (ICL)**: Поддерживает быстрое клонирование голоса по короткому эталону WAV (3–7 секунд) и тексту реплики.
* **Дизайн голоса (Instruct)**: Поддерживает синтез речи по текстовому описанию голоса с использованием специальной модели `voicedesign` — без аудиоэталона, достаточно описать голос (например, `male, adult, moderate pitch`).
* **Автономность**: Не требует подключения к интернету после скачивания моделей.

## Варианты установки и сборки

Вы можете настроить адаптер двумя способами в зависимости от того, нужна ли вам готовая к работе сборка или кастомная сборка из исходников:

* **Готовый релиз (Рекомендуется)**: Скачайте последний архив на вкладке [Releases](). Он уже содержит предварительно скомпилированную библиотеку `qwen.dll`, DLL-файлы вычислительных бэкендов (`ggml-cuda.dll`, `ggml-vulkan.dll`), а также автономные исполняемые утилиты внутри папки `bin/`.
* **Сборка из исходников (Продвинутый уровень)**: Если вы хотите скомпилировать основной движок вручную, этот репозиторий включает в себя `qwentts.cpp` в качестве подмодуля Git. Вы можете подтянуть подмодуль и собрать бинарные файлы прямо на своей машине с помощью CMake и предпочитаемого вами компилятора (MSVC/GCC).

## Быстрый старт

1. **Распаковка и клонирование**: 
   * Если вы используете готовый релиз, просто распакуйте папку `QwenTTS` в вашу рабочую директорию.
   * Если вы собираете проект из исходников, клонируйте этот репозиторий рекурсивно, чтобы подтянуть подмодуль:
     ```bash
     git clone --recursive https://github.com/Holastor/Qwentts-SkyrimNet-Backend.git
     ```
2. **Настройка окружения**: Запустите файл `Setup_QwenTTS_Adapter.bat`. Он создаст локальное виртуальное окружение `.venv` и установит все необходимые зависимости Python.
   > [!NOTE]
   > Скрипт успешно завершится, даже если файлы моделей отсутствуют. Вы сможете скачать их позже через панель управления.
3. **Запуск сервера**: Запустите `Start_QwenTTS_Persistent.bat`. Сервер запустится и автоматически откроет дашборд настроек в вашем браузере по умолчанию.
4. **Настройка SkyrimNet**: В настройках клиента SkyrimNet укажите следующий URL-адрес TTS-сервера:
   ```
   http://127.0.0.1:7861
   ```

## Панель управления (Dashboard)

Адрес панели управления: **[http://127.0.0.1:7861/settings](http://127.0.0.1:7861/settings)**

### 1. Вкладка «Настройки и Модели» (Settings & Models)
* **Backend**: Выбор вычислительного ядра (`CPU`, `CUDA` для карт NVIDIA или `Vulkan` для AMD/интегрированных карт).
* **Параметры генерации**: Регулировка Temperature (выразительность голоса), Repetition Penalty, Seed (зерно генерации) и др.
* **Управление моделями**:
  * Если папка `models/qwen/` пуста, вы можете **скачать модели напрямую из HuggingFace** в один клик в блоке "Download from HuggingFace".
  * Выберите активную языковую модель (`Talker LM`) и аудиокодек (`Audio Codec`) и нажмите **Apply Models** для перезагрузки бэкенда.

### 2. Вкладка «Менеджер голосов» (Voices Manager)
* **Импорт из SkyrimNet**: Синхронизируйте и автоматически скачайте эталоны голосов из запущенного сервера SkyrimNet.
  > [!IMPORTANT]
  > Процесс импорта показывает наглядный **прогресс-бар, процент выполнения, количество обработанных голосов и время до конца**. Перед импортом убедитесь, что в параметрах синтеза выбран правильный язык (например, `russian`), чтобы эталоны разложились по нужным папкам!
* **Ручное управление**: Добавление, редактирование, прослушивание (встроенный аудиоплеер внизу) и удаление голосов NPC из базы данных `voice_refs_<lang>.json`.

### 3. Вкладка «Дизайн голосов» (Voices Design)
> [!NOTE]
> Эта вкладка требует загруженной модели **voicedesign** (например, `qwen-talker-1.7b-voicedesign-Q8_0.gguf`) в разделе Модели.

* **Импорт из SkyrimNet (Voice Design)**: Массовая регистрация всех типов голосов из запущенного сервера SkyrimNet с автоматически сгенерированными текстовыми описаниями (атрибуты пола) — без скачивания WAV-файлов.
* **Ручное управление**: Добавление, редактирование и удаление записей дизайна голосов. Каждая запись состоит из **Ключа спикера** и **Описания голоса** (например, `female, young adult, high pitch`).
* **Предпрослушивание**: Тестирование любого дизайна голоса прямо из панели управления — введите текст для предпрослушивания, нажмите ▶ Слушать, и адаптер синтезирует речь, используя instruct-описание. Язык автоматически берётся из текущих настроек синтеза.

> [!TIP]
> Режим Voice Design и режим ICL (клонирование) используют **раздельные базы данных**: `voice_design_<lang>.json` и `voice_refs_<lang>.json` соответственно. Один и тот же ключ спикера может существовать в обеих базах без конфликтов. Адаптер автоматически маршрутизирует синтез в зависимости от типа загруженной модели.

## Два режима синтеза

QwenTTS поддерживает два принципиально разных подхода к синтезу речи:

| Характеристика | ICL Mode (Клонирование) | Voice Design (Instruct) |
|---|---|---|
| **Модель** | `base` модель (напр., `qwen-talker-1.7b-base-Q4_K_M.gguf`) | `voicedesign` модель (напр., `qwen-talker-1.7b-voicedesign-Q8_0.gguf`) |
| **Вход** | Эталонный WAV-файл + точный транскрипт | Текстовое описание атрибутов голоса |
| **База данных** | `voice_refs_<lang>.json` | `voice_design_<lang>.json` |
| **Вкладка** | Менеджер голосов | Дизайн голосов |
| **Качество** | Высокоточное клонирование конкретного голоса | Стилизованный голос на основе описания |
| **Применение** | Точное соответствие голосу NPC | Быстрое прототипирование, типовые голоса |

> [!IMPORTANT]
> Адаптер автоматически защищает от несовпадения режимов: если загружена модель `voicedesign`, но запрошен пользовательский (ICL) голосовой ключ (или наоборот), система вернёт тишину вместо аварийного завершения.

```
│   └── voice_refs/               # Базы данных голосов
│       ├── voice_refs_ru_RU.json       # Пользовательские голоса (ICL режим)
│       └── voice_design_ru_RU.json     # Дизайны голосов (instruct режим)

│   ├── core/                     # Ctypes-обертка для qwen.dll и бэкенд
│   ├── services/                 # Синтез, кэширование, импорт голосов
│   └── web/                      # Дашборд (settings.html, settings.css, settings.js)
└── output_temp/                  # Временные генерируемые WAV и логи диагностики
```

## Требования к эталонам (ICL Mode)

Для качественного клонирования голоса QwenTTS требует:
1. **Качественный эталонный WAV** (желательно моно, 24 кГц, длительностью **3–7 секунд**).
2. **Точный транскрипт** (параметр `ref_text`). Если текст не совпадает с тем, что сказано на записи, качество синтеза резко падает.

## Диагностика и лог-файлы

В случае подозрительного поведения или ошибок проверяйте лог-файлы:
* Логи синтеза пишутся в стандартный вывод консоли.
* Временные сгенерированные файлы сохраняются в `output_temp/qwentts_generated/`.
* При ошибках генерации отладочная информация сохраняется в `output_temp/qwentts_debug_failed/`.
* Текст, переданный в модель, сохраняется в `output_temp/qwentts_debug_text/`.
